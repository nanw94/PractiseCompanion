# Practice Companion v0 spec

## Product goal
Help a musician start a timed practice session quickly, capture what they did, and review basic history. v0 is local-only (no login) and responsive (mobile + desktop).

## Core experience
- Start a session in one tap
- Pause/resume while practicing
- Finish to log notes/tags quickly
- Review history and today’s summary

## Navigation / IA
- `/` Home
- `/session` Active session (timer)
- `/log` Log session (after finishing)
- `/history` History (list + detail/edit/delete)
- `/settings` Settings (export/import/reset + preferences)

### Global navigation behavior
- If a session is active, show a persistent “Session running” affordance (header chip/button) that goes to `/session`.
- If no session is active, primary CTA is “Start session”.

## Screen specs

### Home (`/`)
**Primary CTA**
- Start session

**Today summary**
- Total minutes practiced today
- Sessions count today
- Top tags today (up to 3)

**Empty state**
- Friendly prompt to start first session

### Active session (`/session`)
**Timer**
- Elapsed time prominently displayed
- Controls: Start (if not started), Pause/Resume, Finish

**Quick log inputs (lightweight, optional)**
- Tags (chip picker + quick add)
- Notes (textarea; autosave draft while session runs)
- Instrument (optional select/text input; can default from settings later)

**Finish**
- Takes the captured data + timestamps to `/log` for confirmation and final edits

### Log session (`/log`)
**Confirm**
- Start/end timestamps and duration (editable duration optional later; v0 read-only)

**Fields**
- Instrument (optional)
- Tags (multi)
- Rating and/or Focus (1–5, optional)
- Notes

**Actions**
- Save (persists session and clears active session)
- Discard (clears active session without saving; confirm dialog)

### History (`/history`)
**List**
- Each row: date, start time, duration, instrument (if set), up to 3 tags

**Filters (v0 minimal)**
- Tag filter (single tag)

**Detail**
- View full session info
- Edit notes/tags/instrument/rating
- Delete (confirm)

### Settings (`/settings`)
**Data**
- Export sessions as JSON file
- Import sessions JSON file (merge with dedupe by id)
- Wipe all data (confirm)

**Preferences (optional for v0, can stub)**
- Default instrument
- Default tags

## UI & component blueprint (Mantine)

### Layout pattern
- Use Mantine `AppShell` as the root frame
- **Header**: app name + “Session running” indicator (when active session exists)
- **Mobile nav**: bottom tabs for Home, Session, History, Settings
- **Desktop**: header-only for v0 (can add side nav later)

### Proposed reusable components
- `AppFrame`
  - Wraps `AppShell`
  - Renders header + responsive navigation
  - Shows “Session running” indicator if an `activeSessionDraft` exists
- `TimerCard`
  - Displays elapsed time
  - Buttons: start/pause/resume/finish
- `TagPicker`
  - Selected tags as pills/badges
  - Add tag by typing and pressing enter (v0)
  - Optional: show “recent tags” based on history
- `SessionSummaryRow`
  - A compact row for history list: date/time + duration + tags

## Route-to-file map (Next.js App Router)

### Routes (new)
- `app/session/page.tsx`: active session timer + quick tags/notes (uses `TimerCard`, `TagPicker`)
- `app/log/page.tsx`: log/confirm screen after finishing (form fields + save/discard)
- `app/history/page.tsx`: session list + filters + detail modal or inline detail
- `app/settings/page.tsx`: export/import/reset + optional preferences

### Existing routes (update)
- `app/page.tsx`: becomes the Home screen (start/continue + today summary)

### Shared modules (new)
- `lib/model.ts`: types for `Session`, `ActiveSessionDraft`, `AppData`
- `lib/storage.ts`: `loadAppData()`, `saveAppData()`, `appendSession()`

### UI components (new, location suggestion)
- `components/AppFrame.tsx`: Mantine `AppShell` + navigation + session indicator
- `components/TimerCard.tsx`
- `components/TagPicker.tsx`
- `components/SessionSummaryRow.tsx`

### State hooks (new)
- `hooks/useActiveSessionDraft.ts`: start/pause/resume/finish draft + persist to storage
- `hooks/useSessions.ts`: load sessions + derived stats (today totals, recent tags)

## Implementation order (recommended)
1. Create `AppFrame` and wire it into `app/layout.tsx` (so all pages share nav)\n+2. Implement `useActiveSessionDraft` and `/session`\n+3. Implement `/log` to finalize and save sessions\n+4. Implement `/history` list and detail\n+5. Implement `/` home summary widgets\n+6. Implement `/settings` export/import/reset\n+
## Non-goals for v0
- Accounts/sync
- Advanced analytics
- Routines/sets builder
- SRS scheduling
